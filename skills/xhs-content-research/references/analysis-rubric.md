## §A 笔记级共用字段（图文/视频都填）

逐篇打标，进 Bitable 素材库。

| 维度 | 取值枚举 | 用途 |
|---|---|---|
| **人设类型** | 素人路人 / 专业达人 / 品牌官方 / KOC测评党 / 生活方式博主 | 判断可信度与模仿成本 |
| **选题角度** | 测评 / 攻略 / 避坑 / 种草 / 教程 / vlog / 情绪共鸣 / 对比 | 看哪类角度高频高赞 |
| **标题公式** | 数字清单 / 悬念留白 / 痛点直击 / 身份代入 / 对比反差 / 提问 / 夸张结果 | 沉淀标题套路库 |
| **首句钩子** | 利益承诺 / 制造焦虑 / 反常识 / 直接提问 / 场景代入 | 看开头怎么留人（图文=首句；视频=前3秒） |
| **封面形式** | 大字报 / 实拍单图 / 拼图九宫 / 前后对比 / 人像出镜 / 纯文字卡 | 喂给 xhs-visual-director 的封面建议来源 |
| **标签策略** | 标签数 + 类型：大词 / 长尾 / 话题 / 品牌 | 看流量打法，判断笔记覆盖的流量入口 |
| **互动数据** | 赞 / 藏 / 评 / 分享 | 判断爆款程度，原始数值录入 |
| **互动结构** | 赞藏比、评论率 | 推断收藏型（教程/攻略）vs 讨论型（情绪/争议）内容 |
| **爆款归因** | 一句话：为什么这篇火 | 人工洞察，最值钱的一栏；无法归因则留空 |

> **赞藏比** = 藏 ÷ 赞；**评论率** = 评 ÷ 赞。两项在 Bitable 可由公式列自动计算，本 rubric 只定义维度名。

## §B 按笔记类型分叉字段

**视频专属字段在图文笔记一律留空**；图文专属字段在视频笔记留空。以 Bitable 的 `笔记类型（图文/视频）` 列区分。

| 图文专属 | 视频专属 |
|---|---|
| **内容结构(图文)**：总分 / 编号清单 / 故事叙事 / step-by-step / 测评打分 | **内容结构(脚本)**：开场钩子 → 痛点 → 解决 → 引导 |
| — | **视频时长**：秒数或时长区间（如 60s / 3-5min） |
| — | **拍摄形式**：口播 / vlog / 教程录屏 / 剧情 / 混剪 |
| — | **节奏元素**：BGM / 卡点 / 字幕条 / 花字（可多选） |
| — | **完播信号**：时长 vs 信息密度的推断（如"时长短+信息密=完播率高"） |

**字段名说明（供后续任务引用）：**
- 图文专属字段名：`内容结构(图文)`
- 视频专属字段名：`内容结构(脚本)`、`视频时长`、`拍摄形式`、`节奏元素`、`完播信号`

## §C 跨篇洞察维度

汇总多篇笔记后提炼，出 markdown 报告，不进 Bitable 逐行字段。

1. **选题机会** — 高频高赞角度（红海） vs 高赞少做（蓝海）；用矩阵或列表呈现，突出值得切入的空白点。
2. **标题套路库** — 把高赞标题归类为 3–5 个可复用公式，附代表性原文示例。
3. **封面套路库** — 高赞封面形式的分布比例，最终给 xhs-visual-director 的推荐封面形式清单。
4. **标签打法** — 头部笔记的标签共性：大词/长尾/话题/品牌的组合比例与高频词。
5. **内容缺口** — 梳理评论区高频诉求中未被已有笔记满足的角度，即可做的新选题。
6. **图文 vs 视频** — 样本里两种形态的占比与各自爆款率，得出"这选题该做图文还是视频"的建议。

## §D 模式特异性标注

三种调研模式在 ANALYZE/OUTPUT 步骤的额外侧重，不改变基础 rubric，只在固定字段之上叠加。

### 选题调研
- 额外计算「角度 × 互动」热力分布：以 `选题角度` 为行，赞/藏/评均值为列，做二维热力矩阵，突出蓝海（高赞少量）区域。
- 报告重点：选题机会（§C 第1项）放首位，给出"可做 / 竞争过热 / 蓝海待验证"的三档判断。

### 竞品账号

竞品分析按**调研焦点（lens）**展开，**不按"商业号/内容号"硬分账号类型**（账号是连续谱、常混合，硬分会逼出错误分类）。GOAL 阶段先确认看哪个焦点：**内容打法 / 商业模式 / 两者都要**。

**Lens A — 内容打法（默认必跑）**：还原「这个号内容怎么做的」。
- **发布节奏**：发布频率（篇/周）、常用发布时段（卡片不显示时间，用 gathering.md 的 `__INITIAL_STATE__` 取数法批量取）。
- **账号爆款率**：爆款篇数 ÷ 总采集篇数（爆款阈值可按赞>500或自定）。
- **选题矩阵**：各选题角度的分布与各自互动，判断内容策略重心。
- **内容 IP 构成**：单人 vs 多人、原创拍摄 vs 长内容切片（决定模仿成本）。

**Lens B — 商业模式（账号有变现层时叠加；纯内容号在报告里注明"未见明显变现层"后跳过）**：还原「这个号怎么赚钱的」。
- **产品矩阵与价格漏斗**：在售商品/课程的价位分层、销量、形态；区分**主推 vs 仅陈列**（按挂载频次/销量判断，别把陈列品当主推）。
- **挂车 × 选题匹配**：逐篇记录笔记挂的商品/课程/链接，归纳「什么内容挂什么货」的匹配规律（挂车=App-only 数据，见 gathering.md）。
- **转化路径**：笔记→成交的多触点链路——挂车、置顶评论 CTA、主页橱窗/简介、私域群、直播；标出哪层做即时转化、哪层做高客单承接。
- 变现机制类结论**必带证据等级**（见下「证据分级」）——这类结论常无法从公开数据直接看到。

- 报告重点：选了哪个 lens 就以哪个为首位（内容打法→账号规律总结；商业模式→变现漏斗拆解）；两者都要时先内容、后商业。

### 单篇拆解
- 对单篇做**逐段/逐镜精拆**：图文逐张图说明"这张在干嘛"；视频逐时间段标注"镜头/文案/节奏"。
- 精拆结果以表格呈现（段落/时间段 | 内容 | 作用/策略），附总结性爆款归因。
- 单篇拆解不强求进 Bitable，可仅出 markdown；若用户需要存档可选择性写入。

### 三模式通用
- 无论哪种模式，报告都必须标注：
  - 样本中图文 vs 视频的数量占比。
  - 图文爆款率 vs 视频爆款率（用同一爆款阈值计算，确保可比）。
  - 此标注是 §C 第6项「图文 vs 视频」洞察的基础数据来源。

### 证据分级（所有跨篇洞察与变现结论必带）
每条 §C 洞察、§D Lens B 变现结论，末尾标一个等级，区分事实与推断：
- `实测`：从采集到的数据直接算出/看到（爆款率、发布时段、某篇实际挂的课）。
- `推断`：数据 + 通行规律的合理推理，但未直接观测到（如"直播间用折扣促单"）。
- `未观测`：相关但本次拿不到（如私域群内话术、商品详情页未核）；只标方向，不写成事实。

规则：薄样本（1–2 个数据点）的规律**必须注明样本量**；下不了判断就写"未观测/待验证"，不靠想象补全。这是为了一处常见返工——把单点观察或行业惯例当成"该账号事实"写进结论。
