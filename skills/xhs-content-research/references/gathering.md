# 采集 Playbook — 小红书笔记采集方式

本文件定义三种采集方式的完整操作步骤，按优先级/降级顺序排列。每种方式说明：怎么做 / 能拿到哪些字段 / 拿不到如何标注。

---

## 通用约定

- **每篇笔记必须记录 `采集方式`**，取值限定为以下三种枚举（供 bitable-schema 引用）：

  | 枚举值 | 含义 |
  |--------|------|
  | `自动` | Playwright MCP 全自动采集 |
  | `手动` | 用户粘贴链接/截图/正文，skill 结构化拆解 |
  | `导入` | 第三方平台（千瓜/新红/灰豚）导出文件 |

- **字段缺失统一留空，不编造**。凡当次采集无法获取的字段，在 Bitable 对应单元格保持空值，不推测或填写占位文字。
- 降级顺序：Playwright 自动化 → 用户手动喂 → 第三方导出。实际采集优先尝试首选方式，遇阻立即降级，不强行绕过。

---

## 方式一：Playwright 自动化（首选）

### 怎么做

使用 Playwright MCP 工具驱动浏览器，全程无需用户手动操作。

**前置条件：** 浏览器会话必须已登录小红书账号（需有效 cookie / session）。若未登录，提示用户先手动登录后再重新触发采集，或直接降级到方式二。

**操作步骤：**

1. **导航到搜索页**
   ```
   browser_navigate → https://www.xiaohongshu.com/search_result?keyword=<关键词>
   ```

2. **等待页面加载，截取初始快照**
   ```
   browser_snapshot → 确认搜索结果列表已渲染
   ```
   若快照显示登录弹窗或滑块验证码，**立即停止并降级**（见「遇阻降级」）。

3. **逐条点进笔记**
   ```
   browser_click → 点击第一条笔记卡片
   browser_snapshot → 获取笔记详情页完整内容
   ```
   从快照中提取以下字段：
   - 标题（`<title>` 或笔记正文首行）
   - 正文全文
   - 标签列表（`#话题` 形式）
   - 公开互动数：赞数、收藏数、评论数、分享数（若页面显示）
   - 封面图 URL（第一张图片的 `src`）
   - 发布日期
   - 账号名
   - 笔记类型（图文 / 视频，根据页面元素判断）

4. **返回列表，采集下一条**
   ```
   browser_navigate_back → 返回搜索结果列表
   ```
   重复步骤 3，直到完成目标样本量。

5. **视频笔记的额外处理**
   视频笔记页面可截取：视频时长（页面显示的时长标注）。脚本/节奏元素无法自动获取，对应字段留空。

### 能拿到的字段

| 字段 | 可采集性 |
|------|---------|
| 笔记链接 | 当前页面 URL |
| 笔记 ID | 从 URL 中提取 |
| 账号名 | 页面元素 |
| 发布日期 | 页面元素 |
| 笔记类型（图文/视频） | 页面元素判断 |
| 标题 | 页面元素 |
| 正文 | 页面元素 |
| 标签 | 页面 `#话题` 元素 |
| 赞数 | 页面公开数据 |
| 收藏数（藏） | 页面公开数据 |
| 评论数（评） | 页面公开数据 |
| 分享数 | 页面公开数据（部分笔记不显示） |
| 封面图 URL | 第一张图片 src |
| 视频时长 | 仅视频笔记，页面标注 |

### 拿不到的字段

以下字段 Playwright 无法自动采集，**统一留空**：

- 粉丝量级（账号主页需单独点进，成本高；保留为空，手动补充）
- 分享数（部分笔记不对外展示，不强行推断）
- 视频脚本结构、节奏元素、完播信号（需人工分析，自动化阶段留空，ANALYZE 步骤人工填）
- 内容结构（图文/脚本）、封面形式、人设类型、选题角度等打标字段（这些是 ANALYZE 步骤的人工标注，不在采集阶段填写）

### 遇阻降级

以下情况**必须立即停止 Playwright 采集，不尝试绕过**，降级到方式二（手动）：

- 页面出现**验证码**（图形验证码、滑块、拼图等任何形式）
- 页面出现**登录弹窗**（session 失效）
- 连续多次 `browser_snapshot` 内容为空或页面内容不可读
- 页面返回明显的**反爬错误提示**（如「访问受限」「请求异常」）

降级时向用户说明：「Playwright 采集遇到反爬/验证码，已停止。请切换到手动方式（粘贴笔记正文/截图）继续采集。」

`采集方式` 标记为 `自动`（已成功采集的笔记保留此标记；未完成的笔记在降级后采集，标 `手动`）。

## 方式二：用户手动喂（降级）

### 怎么做

用户直接提供笔记内容，skill 负责结构化拆解，**不发起任何网络请求**。

**接受的输入格式（任选其一或组合）：**

1. **笔记链接**：`https://www.xiaohongshu.com/explore/<id>` 或 `https://xhslink.com/<短链>`
   - 仅从 URL 中提取笔记 ID，其余字段由用户补充或留空。
2. **截图**：用户上传笔记截图（含标题、正文、标签、互动数）
   - skill 从图像内容中识别可读文字，无法识别的字段留空。
3. **粘贴正文**：用户复制笔记标题+正文+标签文本
   - skill 按结构拆解，互动数等需用户额外提供。

**操作步骤：**

1. 接收用户输入，识别输入格式。
2. 从输入中提取可获取的字段（见下表）。
3. 对用户未提供的字段，**保持空值**，不请求用户补充所有字段（只补核心字段：链接、标题、笔记类型）。
4. 将 `采集方式` 标记为 `手动`。
5. 进入 NORMALIZE 步骤继续处理。

### 能拿到的字段

取决于用户提供内容的完整程度：

| 用户提供内容 | 可获取字段 |
|------------|-----------|
| 链接 | 笔记链接、笔记 ID |
| 截图（含互动数） | 标题、正文片段、标签、赞/藏/评/分享、账号名、发布日期 |
| 粘贴正文 | 标题、正文全文、标签 |
| 用户口头补充 | 任意字段（用户自述即可录入） |

### 拿不到的字段

- 封面图 URL（截图可视为封面图，但无 URL；留空或填「截图」备注）
- 视频时长、粉丝量级等（用户未提供则留空）

`采集方式` 标记为 `手动`，不受反爬影响，零平台风险。

## 方式三：第三方导出（可选）

### 怎么做

用户从千瓜数据、新红数据、灰豚数据等第三方平台导出 Excel 或 CSV 文件，skill 读取后做二次分析。

**操作步骤：**

1. **接收文件**：用户将导出文件路径或内容提供给 skill（文件格式：`.xlsx` / `.csv`）。
2. **读取文件**：使用 Read 工具或解析文本读取文件内容，识别列名与数据行。
3. **字段映射**：将第三方平台的列名映射到本 skill 的字段名（见下方映射示例）。未匹配的列忽略；本 skill 字段在导出文件中无对应列时留空。
4. **批量导入 Bitable**：对每行记录，`采集方式` 标记为 `导入`，通过 `bitable_v1_appTableRecord_create` 批量写入。
5. 进入 ANALYZE 步骤对全量数据做二次分析（打标字段在此步骤人工或 AI 补充）。

**常见第三方平台字段映射示例：**

| 第三方平台字段（示例） | 本 skill 字段 |
|----------------------|--------------|
| 笔记标题 | 标题 |
| 发布时间 | 发布日期 |
| 点赞数 | 赞 |
| 收藏数 | 藏 |
| 评论数 | 评 |
| 转发数 | 分享 |
| 博主昵称 | 账号名 |
| 粉丝数 | 粉丝量级 |
| 笔记类型 | 笔记类型（图文/视频） |
| 笔记链接 | 笔记链接 |

> 不同第三方平台列名不同，映射时以语义对齐为准，不要求字段名完全一致。遇到无法映射的列，忽略即可。

### 能拿到的字段

第三方平台通常提供的数据比 Playwright 更丰富（含粉丝量级、阅读量等付费数据），具体取决于平台套餐：

- 采集元数据：笔记链接、笔记 ID、账号名、粉丝量级、发布日期、笔记类型
- 互动数：赞、藏、评、分享（通常齐全）
- 部分平台额外提供：阅读量、互动率、收录话题、商业合作标注

### 拿不到的字段

- 打标字段（人设类型/选题角度/标题公式/封面形式等）：第三方平台不提供，在 ANALYZE 步骤人工补充，此时留空。
- 封面图 URL：部分平台提供缩略图链接，部分不提供；不提供则留空。

`采集方式` 标记为 `导入`。

---

## 三种方式对比速查

| 维度 | 自动（Playwright） | 手动 | 导入 |
|------|------------------|------|------|
| `采集方式` 枚举值 | `自动` | `手动` | `导入` |
| 反爬风险 | 有（遇阻即停） | 无 | 无 |
| 样本规模 | 中等（受反爬限制） | 小（逐篇） | 大（批量） |
| 互动数据完整性 | 公开数据（4 项） | 依用户提供 | 通常完整（含付费数据） |
| 粉丝量级 | 留空 | 依用户提供 | 通常有 |
| 打标字段 | ANALYZE 步人工补 | ANALYZE 步人工补 | ANALYZE 步人工补 |
| 适用场景 | 快速批量抓取基础数据 | 单篇深度拆解 / 临时调研 | 竞品账号批量分析 |
